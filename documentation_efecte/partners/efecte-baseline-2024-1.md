  Efecte Baseline 2024.1 includes several feature improvements and bug fixes across Efecte solution portfolio. One cross-solution key feature for IGA and Service Management solutions is the capability for provisioning multiple accounts for one person.
The main improvements that apply to the entire Efecte Baseline:

 Provisioning multiple accounts for one person
 Efecte Delivery Management improvements

In Service Management baseline main improvements are:

 Improved Risk Management -solution
 Atlassian Jira Cloud and ESM connectors (ask your sales manager)
 Local user creation -process improvements
 Better interaction between business hours, countries, exception dates/days
 Improved ESS2 catalog structure

In IGA baseline main improvements are:

 Renewed user lifecycle management 
  
   The solution now features an enhanced user lifecycle management process, designed to streamline and improve the overall efficiency of data management.
  
 Support for multiple work periods 
  
   Users can now have multiple active work periods simultaneously, facilitating flexible employment scenarios while ensuring that access rights are accounted for.
  
 Primary work period calculation 
  
   The system calculates a primary work period when a user has multiple work periods active or upcoming. 
  
 Renewed Self-Service forms 
  
   Self-service forms have been redesigned to accommodate multiple work periods and accounts. This update allows users to manage their details more effectively, promoting accuracy and ease of use in updating personal and employment information.
  
 Updated access right management 
  
   Access right management has been overhauled to support multiple work periods and accounts. This ensures that rights and permissions are accurately assigned according to each specific work period.
  
 Out of the box support for three (3) directories 
  
   AD
   Entra ID (Azure AD)
   OpenLDAP
