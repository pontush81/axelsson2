 Problem statement  
 Administrators want to define form fields with custom drop-down values and map the information to a single text attribute.   
  Short description  
 In the Self-service configuration area, the administrator can create a form field, define a custom set of drop-down values, and map the field to the text attribute. The admin is informed while configuring if the text attribute is already used in the multi-mapping configuration.   
  Use case details  
 As an ESS2 admin, I want to present a custom set of options for questions to my end-users that they can select from, to help provide more specfic information about an issue when they are completing a form.  
 AC:   
 
   Admin can choose to define a custom set of values for a drop-down   
   For custom values, the admin can add one or more values to create a list   
   Admin can delete an existing value from the list of values   
   Admin can rename a value   
   Admin can change the order of values   
   Admin can set the target attribute to a  
   
     string (drop down)   
     multivalue string (checkbox/multiselect dropdown).   
    
   When populating the form in the Self Service Portal, the custom values are visible as selectable options
