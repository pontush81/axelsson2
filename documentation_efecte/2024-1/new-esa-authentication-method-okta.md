As a customer of Efecte's Secure Access tool (ESA), I want to use Okta authentication. 
Background information:
What?

 Okta authentication is asked by multiple customers. 

How?

 ESA is to be able to login users from Okta.

Why?

 ESA Authentication catalog must be comprehensive for customers.
 Manually entering the username and password for ESA is time-consuming and unproductive. SSO streamlines the user validation process for login and makes it easier to access protected resources.
 Enhanced Security: Okta provides advanced security protocols and features such as multi-factor authentication. These measures help protect user accounts from unauthorized access and ensure data integrity.

To whom?

 All ESA customers 
  
   ESA Administrator can decide to use or not to use
